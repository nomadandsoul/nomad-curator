Delayed Shipment Setup

Template WhtNFA is live. Dit document beschrijft het volledige Make.com scenario dat de proactieve vertraging-mail triggert — precies te doen om 10:15u.
Architectuur — hoe het werkt
Doel: Elke bestelling die langer dan 5 dagen (NL/BE) na fulfillment geen bezorgbevestiging heeft gekregen, triggert automatisch een Klaviyo-event. Klaviyo stuurt dan template WhtNFA.

Reden: Shopify geeft alleen bezorgbevestiging voor bepaalde vervoerders. Bij dropshipping (ibizamode, Droppery) ontbreekt dit vaak. Make.com vult dit gat op met een dagelijkse check.
1
Shopify → Make.com: Elke dag om 09:00 checkt Make.com alle orders die meer dan 5 dagen geleden fulfilled zijn zonder Delivered Shipment event in Klaviyo.
2
Make.com → Klaviyo: Voor elke gevonden bestelling stuurt Make.com een custom event Shipment Delayed naar Klaviyo met ordernummer, datum en tracking URL.
3
Klaviyo → klant: Een nieuwe flow getriggerd door Shipment Delayed stuurt direct template WhtNFA. Eenmalig per order.
Make.com — Scenario stap voor stap
Verbinding: Nomada Live (nomadandsoul.nl)
1
⏰ Schedule

Module: Schedule
Type: Every day · Tijd: 09:00 · Tijdzone: Europe/Amsterdam
2
🛍️ Shopify

Module: Shopify → Search Orders
Verbinding: Nomada Live (nomadandsoul.nl)
Filter: fulfillment_status = fulfilled
Filter: created_at < now minus 5 days
Limit: 50 (pagineer indien nodig)
In Make.com: gebruik de Shopify → Search Orders module. Zet als filter:
Field: fulfillment_status · Operator: equal to · Value: fulfilled
En: updated_at · Operator: less than · Value: {{addDays(now; -5)}}
3
🔀 Router

Module: Router
Voeg een Iterator toe om elke order afzonderlijk te verwerken. Koppel de Shopify-uitvoer aan de iterator.
4
📡 HTTP

Module: HTTP → Make a request (Klaviyo Events API)
URL: https://a.klaviyo.com/api/events/
Method: POST
Headers:
  Authorization: Klaviyo-API-Key YOUR_PRIVATE_KEY
  Content-Type: application/json
  revision: 2024-02-15

Body (JSON):
{ "data": { "type": "event", "attributes": { "metric": { "data": { "type": "metric", "attributes": { "name": "Shipment Delayed" } } }, "profile": { "data": { "type": "profile", "attributes": { "email": "{{iterator.email}}" } } }, "properties": { "OrderNumber": "{{iterator.name}}", "OrderDate": "{{formatDate(iterator.created_at; 'D MMMM YYYY')}}", "TrackingURL": "{{iterator.fulfillments[].tracking_url}}", "DaysDelayed": 5 } } } }
Private API key: in Klaviyo → Account → Settings → API Keys → Create Private Key (scopes: Events write). Nooit opslaan in Claude-geheugen — in Make.com als versleutelde variabele bewaren.
5
⚙️ Filter

Voeg een Filter toe tussen stap 2 en 4:
Conditie: iterator.financial_status = paid
En: iterator.fulfillments[].tracking_company is NOT empty
Dit voorkomt dat testorders of onbetaalde orders een vertraging-mail krijgen.
6
✓ Gereed

Scenario activeren: Zet het scenario op On. Make.com draait het elke dag om 09:00 en triggert automatisch Klaviyo voor vertraagde bestellingen.
Klaviyo — Delayed Shipment flow (om 10:15u bouwen)
1
Flows → Create Flow → Create From Scratch
Naam: Delayed Shipment
2
Trigger: Metric → Shipment Delayed
Dit event verschijnt in Klaviyo zodra Make.com het eerste keer verstuurt. Daarna is het selecteerbaar.
3
Flow Filter: What someone has done → Received Email matching Delayed Shipment → 0 times since starting this flow
Dit voorkomt dubbele mails bij meerdere checks.
4
Sleep direct een Email onder de trigger → selecteer template WhtNFA
Geen time delay — de mail gaat direct na het event.
5
Klik de email aan → Sending Options → Smart Sending: OFF
Dit is een transactionele melding — Smart Sending mag deze niet blokkeren.
6
