Delayed Shipment — definitieve opzet

Geverifieerd tegen de echte Klaviyo-metrics van dit account: Delivered Shipment bestaat niet (vervoerders geven geen delivery-tracking door aan Shopify). Fulfilled Order bestaat wél al, native. Daarom: geen Make.com nodig voor de basisversie.
Aanbevolen opzet — 100% native Klaviyo Geen externe tools
Waarom dit beter is dan Make.com: zonder een delivered-signaal kan de flow toch niet conditioneel bepalen "is het al bezorgd" — dus is een vaste 5-dagen-vertraging na fulfillment net zo effectief als een custom event, maar zonder API-key, JSON-payload of externe afhankelijkheid. Minder onderdelen, minder foutkans.
1
Flows → Create Flow → Create From Scratch
Naam: Delayed Shipment
2
Trigger: Metric → Fulfilled Order
Native Shopify-metric, synct al automatisch — geen setup nodig.
3
Sleep een Time Delay → 5 days onder de trigger
4
Sleep een Email eronder → selecteer template WhtNFA
5
Klik de email aan → Sending Options → Smart Sending: OFF
6
Review and Turn On
Nog te verifiëren: template WhtNFA gebruikt {{ event.OrderNumber }} en {{ event.TrackingURL }} — deze veldnamen zijn geschreven voor een custom event en zijn nog niet bevestigd voor het Fulfilled Order-formaat. Na de eerste live trigger: open de template in de flow → Preview → klik op de event-variabelen om de exacte veldnaam te bevestigen, en pas de tags aan indien nodig. De template heeft al een fallback ({% if event.TrackingURL %}...{% else %}Binnenkort beschikbaar{% endif %}), dus een ontbrekend veld breekt de mail niet — toont alleen minder detail.
Fallback — Make.com + Events API Alleen nodig bij delivery-tracking
Bewaar dit voor het moment dat jullie een vervoerder toevoegen die wél delivery-bevestiging aan Shopify doorgeeft. Dan wordt een conditionele flow (wél/niet bezorgd) zinvol, en heb je een eigen event nodig om dat te detecteren.
📡 HTTP — Klaviyo Events API (niet de verouderde Track API)
Let op: https://a.klaviyo.com/api/track/ is de legacy v1/v2 API, door Klaviyo zelf uitgefaseerd per 30 juni 2024. Gebruik altijd https://a.klaviyo.com/api/events/ met een private API key in de Authorization-header.
{ "data": { "type": "event", "attributes": { "metric": { "data": { "type": "metric", "attributes": { "name": "Shipment Delayed" } } }, "profile": { "data": { "type": "profile", "attributes": { "email": "{{iterator.email}}" } } }, "properties": { "OrderNumber": "{{iterator.name}}", "OrderDate": "{{formatDate(iterator.created_at; 'D MMMM YYYY')}}", "TrackingURL": "{{iterator.fulfillments[].tracking_url}}", "DaysDelayed": 5 } } } }
🛍️ Make.com Shopify-modules — correcte namen
Make's Shopify-app heeft geen "Watch Fulfillments"-trigger. Beschikbare trigger-modules: Watch Products, Watch Orders, Watch Events, Watch Events (Advanced). Voor dit scenario: Schedule (dagelijks 09:00) → Shopify Search Orders (fulfillment_status=fulfilled, updated_at < now-5d) → Filter (financial_status=paid, tracking niet leeg) → Iterator → HTTP POST hierboven.
Een module die een scenario 5 dagen laat "slapen" (Sleep) bestaat niet realistisch in Make — scenario's hebben executietijd-limieten. Vandaar de Schedule+polling-opzet, niet een enkele wachttrigger.
